# トモリエ（Tomorie）UI/UX 設計書

## 1. 認証機能の更新

### 1.1 認証方式

- **パスワードレス認証**: メールアドレス + 認証コード（6 桁）
- **SSO 対応**: Google, LINE, Facebook
- **セキュリティ**: 2 段階認証（オプション）

### 1.2 認証フロー更新

```
1. メールアドレス入力 → 認証コード送信
2. 認証コード入力（6桁）→ ログイン成功
3. または SSO ボタン選択 → 外部認証 → ログイン成功
```

## 6. シニア向け UI/UX ガイドライン適用例

### 6.1 フォント・色彩設計

```css
/* フォント設定 */
.primary-text {
  font-size: 18px;
  line-height: 1.6;
  font-family: 'Noto Sans JP', sans-serif;
  font-weight: 500;
}

.secondary-text {
  font-size: 16px;
  line-height: 1.6;
  color: #666666;
}

/* ボタン設計 */
.primary-button {
  min-height: 44px;
  padding: 12px 24px;
  border-radius: 8px;
  font-size: 18px;
  font-weight: 600;
}

/* 色彩設計 */
:root {
  --primary-color: #4a90e2; /* 落ち着いたブルー */
  --secondary-color: #f5a623; /* 温かみのあるオレンジ */
  --background-color: #fafafa; /* 淡いグレー */
  --text-color: #333333; /* 濃いグレー */
  --border-color: #e0e0e0; /* 境界線 */
}
```

### 6.2 レスポンシブ対応

- **スマートフォン**: 320px〜767px
- **タブレット**: 768px〜1024px
- **PC**: 1025px 以上
- **文字サイズ**: デバイス設定に連動

### 6.3 アクセシビリティ対応

- **コントラスト比**: WCAG AA レベル（4.5:1 以上）
- **タッチターゲット**: 最小 44px×44px
- **音声読み上げ**: aria-label 対応
- **キーボード操作**: 全機能対応
